  1. 什么是MVVM?

    model view viewModal，是一种设计思想，model,view通过viewModel连接，viewModel通过双向数据绑定把model和view连接起来，通过数据驱动视图。

  2. mvvm 与 mvc 区别？它和其它框架（jquery）的区别是什么？哪些场景适用？

    MVVM和MVC差别不大，都是一种设计思想，controller演变成viewModel,MVVM主要解决MVC中大量的DOM操作，是页面渲染性能降低，加载速度变慢，影响用户体验。
    MVVM是用过数据驱动视图，而不是通过DOM节点操作。
    数据操作比较多的场景。

  10. vuex相关

    • vuex是什么?怎么使用?什么场景下使用

      vuex是vue框架中的状态管理，在main.js里面注入store(包含state，action)，全局的状态使用(登录信息等)

    • vuex有哪几种属性

      state, getters, action, mutation, Module

    • vuex的State特性

      store相当于一个仓库，state就是一个仓库里面对象。通过mapState把state和getters映射到computed(计算属性，计算的值会缓存到内存中，如果计算的数据不会变化，就直接取之前的返回值)中

    • vuex的Getter特性

      getters可以对state进行计算操作，他就是store的计算属性
      多个组件使用，单个组件可以不用

    • vuex的Mutation特性

      action提交的是mutation，用于修改state

    • vuex的action特性

      用于异步处理state，然后通过mutaion修改state

  31. 异步更新队列 - $nextTick()

    vue中的在观察到数据的变化并不是直接去更新DOM，而是开启一个异步队列，并缓冲同一事件发生的所有数据改变。
    例如写一个循环，一百次数据变化，他只会应用于最后一次数据变化，如果没有这种机制，DOM就会去重绘一百次
    一个回调函数

  33. vue 性能优化

    • keys
    • 长列表性能优化：使用Object.freeze()冻结数据，不用数据劫持
    • data函数：不用对象，使用对象的时候，所有实例都会共享一份数据对象，使用函数，没创建一个新的实例后，返回一个新的数据对象，每个组件的实例互不影响，各自独立。
    • $on(‘hook:’): 使用hook + 生命周期的方式绑定
    • 组件懒加载： vue-laze-component
    • 使用事件代理：把事件绑定在父节点
    • 函数式组件：只用来展示数据
    • 路由懒加载：vue-router中配置require,resolve、import(‘组件路径’), require.ensure(官方推荐)，都是按路由划分模块

  34. vue 构建优化

    • Coverage： 谷歌浏览器的新功能，检测代码运行时的页面使用情况和大小（已使用和未使用）
    • tree shaking： 去除无用代码
    • 懒加载：webpack自带的require.ensure
    • webpack优化：
      • webpack-bundle-analyzer： 打包查看资源树，大小
      • productionSourceMap:false
      • 关闭prefetch
      • 组件按需加载: element-ui, ant-design-vue(使用babel-plugin-import)
      • compression-webpack-plugin： 压缩代码（自定义规则）
      • externals： 引入外部文件代替当前。（cdn）
