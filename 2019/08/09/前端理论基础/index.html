  1. html顶部 是什么意思?

    • 区分标准模式和严格模式(标准模式能兼容IE6之前)

    • HTML4.01中有三种HTML类型（Strict，Transitional，Frameset），三种XHTML类型（Strict，Transitional，Frameset），除 frameset 元素取代了 body 元素之外，Frameset DTD 等同于 Transitional DTD，
      一般都会使用Strict
    • 目前现在都使用

  2. html行内元素和块级元素，区别

    • 行内元素(a,b,span,img,input,strong,i,lable,li,td…)
    • 块级元素(body,select,textarea,h1~h6,table,button,p,div,ol,dl…)
    • 块级元素总是在新行开始，占据一整行，宽度自动填满父级，垂直向下排列
    • 块级元素可以设置宽高，外边距，内填充
    • 块级元素可以包含行内元素和其他块级元素
    • 行内元素都会在同一水平线上，溢出自动换行，左右排列
    • 行内元素不可以设置宽高，宽高随文本的变化（行高），外边距上下无效，左右有效，内填充随意
    • 行内元素只能容纳行内元素，或者其他行内元素

  3. HTML5新特性

    • 新语义元素（标签）：header，footer，nav（导航），aside（侧边栏），article（文章），section，hgroup（h1~h6），有利于代码可读性和SEO
    • 增强型表单: date, email, number, range, tel, url …（用于表单录入）
    • 图像处理： aduio, video（视频音频）
    • canvas: 画布（图形容器），可以用js制作复杂动画，甚至可以实时视频处理和渲染。
    • 本地存储： 新增SessionStorage、LocalStorage和IndexedDB加强本地存储，SessionStorage基于会话，关闭浏览器之后存储消失，LocalStorage最多为2.6M，IndexedDB最多为250M。
    • 位置： 新增了Geolocation API访问地理位置，通过window.navigator.geolocation来实现访问，三个方法： getCurrentPosition(), watchPosition(), clearWatch();
    • WebWorker: 当在 HTML 页面中执行脚本时，页面的状态是不可响应的，直到脚本已完成。运行在后台的 JavaScript，独立于其他脚本，不会影响页面的性能。您可以继续做任何愿意做的事情：点击、选取内容等等，而此时 WebWorker在后台运行。
    • WebSocket: WebSocket是HTML5开始提供的一种在单个 TCP 连接上进行全双工通讯的协议，数据可以在客户端和服务端互相发送。

  5. BFC 的作用

    • BFC ： Block Formatting Context（块级格式）
    • 是页面上一个隔离的独立容器，容器内的子元素不会影响到外面的元素。有一个独立的块级渲染区域，该区域有一套渲染规则来约束盒子的布局，与外部的区域没有关系。
    • 实现方式：
      • float不为node
      • position: absolute或fixed
      • overflow: 不为visible
      • display: 不为inline-block、flex、inline-flex
    • 特性:
      • 内部的box会垂直向下排列
      • box垂直方向的高度有margin决定，属于同一个BFC的两个相邻box的margin会重叠
      • BFC的区域不会与浮动区域的box重叠
      • BFC是一个独立的容器，外部的元素不会影响BFC内部的元素，BFC的子元素不会影响外部元素
      • 计算BFC高度是，浮动元素也会参与计算
    • 作用:
      • 取消盒子的margin塌陷
      • 清除内部浮动，可以阻止外部元素被浮动元素覆盖

  7. 什么是 css reset

    • 它的全称就是 reset.css ，是一个 css 文件，用来强行重置浏览器默认的 css 样式。
    • 基本上不用了，现在一般使用Normalize.css，
      • 保护有用的浏览器默认样式而不是完全去掉它们
      • 一般化的样式：为大部分HTML元素提供（这个用的挺多，表单，表格等等）
      • 修复浏览器自身的bug并保证各浏览器的一致性（预格式化文字的font-size，各个浏览器的表单操作等等）
      • 优化CSS可用性：用一些小技巧
      • 模块化，可以自己选择使用的部分和不使用的部分

    • js： 用js代码判断窗口大小，展示不同大小的图片（用处很小）

  13. 移动端适配

    • 关于移动端适配，你必须知道的
    • flexible: flexible方案是阿里早期开源的一个移动端适配解决方案，引用flexible后，我们在页面上统一使用rem来布局。由于viewport单位得到众多浏览器的兼容，上面这种方案现在已经被官方弃用
    • vh、vw: vh、vw方案即将视觉视口宽度 window.innerWidth 和视觉视口高度 window.innerHeight 等分为 100 份
    • viewport：device-width就等于理想视口的宽度，所以设置width=device-width就相当于让布局视口等于理想视口。由于initial-scale = 理想视口宽度 / 视觉视口宽度，所以我们设置initial-scale=1;就相当于让视觉视口等于理想视口，这时，1个CSS像素就等于1个设备独立像素，而且我们也是基于理想视口来进行布局的，所以呈现出来的页面布局在各种设备上都能大致相似

  15. 移动端图片模糊

    • 理论上，位图的每个像素对应在屏幕上使用一个物理像素来渲染，才能达到最佳的显示效果。而在dpr > 1的屏幕上，位图的一个像素可能由多个物理像素来渲染，然而这些物理像素点并不能被准确的分配上对应位图像素的颜色，只能取近似值，所以相同的图片在dpr > 1的屏幕上就会模糊
    • 使用响应式图像

  17. css 兼容问题

    • 浏览器默认样式不同，可以使用Normalize.css，或者自己全局处理

  18. 如何优化网站的SEO

    • 网站结构布局优化：尽量简单, 提倡扁平化结构. 一般而言，建立的网站结构层次越少，越容易被“蜘蛛”抓取，也就容易被收录
    • img标签必须添加“alt”和“title”属性，告诉搜索引擎导航的定位，做到即使图片未能正常显示时，用户也能看到提示文字
    • 把重要内容HTML代码放在最前搜索引擎抓取HTML内容是从上到下，利用这一特点，可以让主要代码优先读取，广告等不重要代码放在下边
    • 控制页面的大小，减少http请求，提高网站的加载速度
    • 合理的设计title、description和keywords
    • 语义化书写HTML代码，符合W3C标准尽量让代码语义化，在适当的位置使用适当的标签，用正确的标签做正确的事。让阅读源码者和“蜘蛛”都一目了然
    • a标签：页面链接，要加 “title” 属性说明，链接到其他网站则需要加上 el=”nofollow” 属性, 告诉 “蜘蛛” 不要爬，因为一旦“蜘蛛”爬了外部链接之后，就不会再回来了
    • 图标使用IconFont替换
    • 使用CDN网络缓存，加快用户访问速度，减轻服务器压力
    • 启用GZIP压缩，浏览速度变快，搜索引擎的蜘蛛抓取信息量也会增大
    • SSR技术
    • 预渲染技术

  20. Linux 754 介绍

    linux文件权限，这串字符可以分成4段理解，结构为“d + 文件所有者操作权限 + 文件所有者所在组操作权限 + 其余人的操作权限”：

    • 1：执行权限
    • 2: 写入权限
    • 4：读取权限
    • 7：表示文件所有者的权限 4 + 2 + 1 = 7
    • 5：表示文件所有者所在组的权限， 4 + 1 = 5

  21. 介绍冒泡排序，选择排序，冒泡排序如何优化

    冒泡排序：从第一位开始，比较两个相邻的大小，按排序方式调换位置，一直到最后。

    选择排序：找一个基准点，比如数组中间的值，根据这个中间值，把小于的放到左边的数组，大于的放到右边的数组，使用回调，直至排序完成

    • 冒泡排序优化

      判断当没有产生交换时，说明排序已经完成，手动断开

  22. transform动画和直接使用left、top改变位置有什么优缺点

    • 定位的left，top会引起页面上其他元素的移动和形状，会影响浏览器产生新的布局，布局的开销巨大，会影响页面的性能。
    • 基于css的动画和原生支持的web动画，通常在称为合成器线程的线程上处理（GPU），减少页面的重绘和回流，性能更佳

  24. 前端优化

    • 减少请求数量: 图片、缓存（304）、接口合并
    • 减少资源大小：压缩、gzip、
    • 优化网络连接: DNS预解析
    • 优化资源加载: 图片服务器、资源加载位置和时机，懒加载和预加载
    • 减少重绘回流： 避免使用层级较深的选择器，css表达式（重绘），元素适当地定义高度或最小高度（回流），css动画（GPU渲染）
    • DOM优化： 减少DOM深度及DOM数量
    • 构建优化：webWorker(兼容)，webpack优化（公共代码，动态导入和按需加载（js的import()，webpack自身的require.ensure），剔除无用代码（tree shaking(uglify js)））。
